Processor State Evolution

Mapping the transition from CPSR (AArch32) to PSTATE (AArch64)

AArch32: The Monolithic CPSR

The **Current Program Status Register ($\text{CPSR}$)** contained all status, control, and privilege information in a single 32-bit register.

CPSR Bit Field Layout (32 Bits)

31 NZCV Bits [31:28]
27 8 Reserved / State / Control (J, T, I, F, A)
4 0 MODE Bits [4:0]

AArch32 Processor Modes (Bits [4:0] Encoding)

Binary Value Mode Name Description
0b10000 User Unprivileged application code (EL0 Equivalent).
0b10011 SVC Supervisor (OS Kernel) Mode.
0b11010 FIQ Fast Interrupt Request.

AArch64: Distributed State (PSTATE)

The **Processor State ($\text{PSTATE}$)** is an abstraction, with status and control functions distributed across dedicated System Registers for cleaner, more secure state management.

Condition Flags ($\text{NZCV}$ Register)

Flags are accessed separately using $\text{MRS}$ and $\text{MSR}$ system instructions.

31 NZCV Bits [31:28]
27 0 Reserved / Other PSTATE Fields

Exception Levels (ELs) Hierarchy

Privilege is defined by the $\text{CurrentEL}$ field (part of $\text{PSTATE}$), providing four clear, distinct levels.
